Triple-Layer Validation
Heading:
Three Chances to Catch Critical Issues
Visual: Three-layer diagram showing validation at each checkpoint
Layer 1: Input Validation
When: During data entry in client questionnaire
What It Catches:

- Missing required fields
- Impossible values (dates, percentages, ages)
- Mathematical conflicts
- Logical inconsistencies
- Regulatory timeline violations
  Advisor Experience:
  Inline error messages appear immediately
  Red highlights on problematic fields
  Suggested corrections offered
  Issues resolved during client conversation, not discovered later
  Example:
  Advisor enters ISO exercise date: 2020-01-15
  System shows grant date: 2021-06-01
  → Immediate error: "Exercise date cannot be before grant date"
  → Advisor corrects in real-time with client

Layer 2: Processing Validation
When: During 40-second report generation
What It Checks:

- All 73 rules executed
- Cross-validation between calculations
- Range checks on computed values
- Consistency across report sections
  System Response:
  Critical failures → STOP generation, return error
  Warnings → Document in Warnings Report
  Both → Log in validation results

Layer 3: Output Validation
When: Final sweep before advisor review
What It Confirms:

- Report consistency maintained
- All warnings properly documented
- No critical issues slipped through
- Audit trail integrity complete
  Safety Net:
  Final check catches edge cases before advisor seesContinue11:11 AMreport.
  Multiple validation stages reduce error probability exponentially.

## Phase 1: Client Engagement

**Badge:** "1"

**Heading:**
Smart Questionnaire with Real-Time Validation (30-60 minutes)

**What Happens:**
Advisor conducts structured questionnaire and conversation with client, capturing
demographics, financial position, niche details, goals, and concerns.

**Real-Time Input Validation:**
As advisor enters data, system validates immediately:
✓ Missing fields: "ISO grant date required for calculations"
✓ Impossible values: "Exercise date cannot be before grant date"
✓ Math conflicts: "Total assets cannot be less than liabilities"
✓ Inconsistencies: "Income below poverty line with luxury assets"
✓ Regulatory violations: "83(b) election must be filed within 30 days"

**Why This Matters:**
Traditional: Discover data issues days later → Must re-contact client → Delays
VelocityIQ: Issues flagged during conversation → Addressed immediately →
Complete data captured in one meeting

**Time Saved:** Hours or days of back-and-forth eliminated

**Privacy:**
Client data used to generate report but NOT stored permanently. No PII retained.

**Visual:** Questionnaire screenshot showing green checkmarks on validated fields, red error with helpful suggestion

## Phase 2: Prompt Generation

**Badge:** "2"

**Heading:**
8-Step Wizard Processes Data (2-3 minutes, automated)

**What Happens:**
Wizard executes automatically, transforming raw client data into comprehensive
generation instructions.

**The 8 Steps (expandable/accordion):**
Step 0: Data Integration
Combines questionnaire + conversation notes into coherent narrative
Step 1: Client Profile Extraction
AI analyzes narrative, extracts key details (name, industry, valuation, goals)
Step 2: JavaScript Parser
Deterministic code structures data (no LLM), builds JSON object
Step 3: Strategic Questions Generation
AI creates 10 advisory questions tailored to client
Step 4: Perplexity Query Generation
Creates 8-10 research queries for live intelligence
Step 5: Research + Narrative Extraction
Calls Perplexity API, extracts family context, combines with profile
Step 6: Framework Selection
Identifies 3-5 relevant reporting frameworks (Tax, Exit, Risk, Estate, etc.)
Step 7: Final Sections Generation
Creates complete report structure, RAG queries, customization notes
Step 8: Payload Assembly + Validation Rules
Merges all steps, loads 73 validation rules, returns complete payload

**Output:**
Complete JSON payload containing client profile, strategic questions, Perplexity
intelligence, family context, RAG queries, report structure, and 73 validation rules.

**Time:** 2-3 minutes fully automated

**Visual:** Flowchart showing 8 steps with data flowing through each stage
